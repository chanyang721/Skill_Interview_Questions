### Node.js가 무엇인가요 ? 
* Nods.js는 V8엔진과 libuv를 이용하여 웹 서버에서 JavaScript를 비동기적으로 사용할 수 있도록 해주는 런타임 환경(프로그램)입니다.
* Non-blocking I/O와 단일 스레스 방식으로 이벤트 루프를 통한 높은 처리 성능을 가지고 있는게 특징입니다.

### Node.js를 사용하는 이유 ?

### node.js의 이벤트 루프가 뭔가요. nodejs에서 비동기 처리를 그림으로 그려서 설명할 수 있나요?
* 이벤트 루프는 싱글 스레드 기반 언어인 자바스크립트에 동시성을 부여하기 위한 개념입니다. 
* 비동기적인 작동처럼 보이는 동시성을 웹 애플리케이션에 부여하기 위해 자바스크립트의 V8엔진에 있는 Call Stack이라는 하나의 스레드와 node.js에서 비동기를 구현하기 위한 libuv 라이브러리를 통해 동시성을 부여하며, 이벤트 루프는 비동기 요청을 마치고 Queue에서 대기중인 콜백함수를 Call Stack으로 우선 순위에 따라 넣어주는 역할을 하기 때문에 node.js에서 동시성을 부여하기 위한 핵심적인 기능을 하고 있습니다.

### 프로세스와 스레드 차이는 무엇인가요.
* 프로세스는 메모라 상에서 실행중인 프로그램을 말하며, 스레드는 이 프로세스 안에서 실행되는 흐름의 단위입니다.

### Node가 싱글 스레드인데 멀티 스레드보다 빠른 이유를 설명해주세요.
* 싱글 스레드는 동기화 작업을 하지 않아도 되기 때문에 멀티 스레드보다 빠릅니다.
* 멀티스레드를 사용하는 경우 공유 자원으로 인한 문제 해결을 위해 동기화에 신경을 써야합니다. 하지만, 싱글 스레드는 이 동기화에 신경쓰지 않아도 되기 때문에 멀티 스레드보다 빠르다고 알고 있습니다. 

### 싱글 스레드와 멀티 스레드 차이점은 무엇인가요.
* 싱글 스레드와 멀티 스레드의 차이점은 동기적, 비동기적인 작업 흐름입니다. 
* 스레드는 프로세스 내부의 작업 흐름의 단위입니다. 따라서 싱글스레드는 작업 흐름이 하나이며, 멀티스레드는 스레드의 개수 만큼 가지고 있습니다. 
* 작업의 흐름의 단위가 하나인 싱글 스레드는 하나의 작업이 끝난 뒤에 두번째 작업을 시작하는 동기적 흐름을 가지고 있는 반면, 멀티 스레드는 스레드의 개수만큼 작업을 비동기적으로 진행할 수 있다는 차이점이 있습니다.
* 하지만, 속도에서는 큰 차이가 없게 되는데요. 작업하는 흐름의 단위만 다를뿐 작업 속도는 큰 차이가 없기 때문에 실제 작업이 완료되는 속도는 비슷하다고 알고 있습니다.

### package.json의 역할은 무엇인가요?
* 해당 프로그램을 실행시키는데 필요한 npm의 모듈과 버전을 명시해 놓은 파일로 협업 시 반드시 필요한 모듈을 정확하게 명시합니다. 이는 npm install 명령어로 손쉽게 프로그램을 실행하기 위한 모듈을 다운받을 수 있도록 하는 역할을 하고 있습니다.
* package.json의 중요한 구조로는 dependencies와 devDependencies 그리고 srcipts가 있습니다. 
* dependencies에는 프로그램 동작에 반드시 필요한 모듈들을 넣고, devDependencies에는 프로그램 동작에 영향을 주지 않지만 필요한 모듈들을 넣습니다. 
* srcipts에는 CLI 환경에서의 명령어를 작성하여 "npm + 명령어"를 입력할 시 해당 명령어의 value로 작성한 문자열을 실행됩니다.
