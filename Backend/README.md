### Get,Post 메소드 차이점 (중요)
* Get과 Post의 차이점은 HTTP 요청을 통해 데이터베이스의 값의 변경 유무라고 할 수 있습니다. 
* Get 메소드는 HTTP 요청 시 단순히 요청에 의한 값을 받아기 위해 사용되지만, Post 메소드는 이와는 반대로 반드시 HTTP 메시지의 Body 부분에 추가하고자 하는 값을 첨부하여 서버단에 있는 저장소에 해당 데이터를 추가하기 위한 목적으로 사용되는 메소드입니다.

### HTTP 와 HTTPS 차이점 (중요)
* HTTP와 HTTPS의 가장 큰 차이점은 HTTP 요청 메시지에 암호화를 적용 했는지 안했는지 여부입니다.
* HTTP는 클라이언트와 서버가 데이터를 주고 받기 위한 프로토콜 중 하나입니다. HTTP는 서버에 암호화 되지 않은 요청 메시지를 서버로 전달합니다. 이런 경우 데이터가 날것 그대로 전달되기 때문에 중간에 요청 메시지가 유출 될 위험이 존재합니다.
* HTTPS는 HTTP에 암호화가 추가된 프로토콜로 데이터를 암호화하여 만약 데이터가 중간에 유출되어도 복호화키가 없다면 어떤 데이터인지 알 수 없도록한 방식입니다. 
* HTTPS는 암호화를 위해 인증서와 한쌍의 복호화 키를 가지고 진행됩니다. 서버가 클라이언트로부터 요청을 받는다면 인증서, 응답 객체, 공개키를 클라이언트에게 보내게 됩니다. 응답 받은 클라이언트에서는 인증서에 적히는 도메인이 자신이 보낸 도메인과 같은지를 비교하여 데이터가 중간에 다른 도메인을 거쳐서 자신에게 도착했는지 확인할 수 있게 됩니다.
* HTTPS는 공개키와 비밀키 개념을 기반으로 하는 비대칭 키 암호화 방식을 사용합니다. 클라이언트와 서버 각각 한쌍의 키를 가지고 있고, 만약 클라이언트가 서버에게 메시지를 보내고 싶은 경우 hand shake단계에서 각자의 공개키를 서로에게 전달 받게 됩니다. 그 후, 전달 받은 공개키로 데이터를 암호화하여 http메시지를 서로에게 보내게 됩니다. 서버는 자신의 공개키로 암호화된 클라이언트의 메시지를 개인키로 복호화하여 전달 받은 메시지를 읽을 수 있게 되는 단계로 진행 됩니다.
